# Skincare Architect

**Skincare Architect** — це інтелектуальний асистент для аналізу косметичних інгредієнтів. Проєкт допомагає користувачам будувати безпечну рутину, уникаючи конфліктів активних речовин (наприклад, ретинолу та кислот) та автоматично розраховуючи послідовність нанесення засобів.

---

## Key Features & Functionality

* **Compatibility Diagnostic**: Ядро системи, що аналізує склад засобів на предмет критичних конфліктів (наприклад, ретинол + кислоти).
* **Parallel Data Processing**: Миттєва обробка великих датасетів (8000+ позицій) з використанням мультипоточності.
* **Smart Routine Builder**: Автоматичне сортування засобів на ранок/вечір та визначення черговості нанесення за текстурою.
* **Personal Collection (Virtual Shelf)**: Система авторизації та менеджменту власної косметички з використанням Persistent Storage (`localStorage`).
* **Global Catalog**: Повнотекстовий пошук по базі даних Sephora з фільтрацією нерелевантних категорій (haircare, makeup тощо).

---

## Bonus Features

1.  **Unit Testing**: Повне покриття бізнес-логіки, стратегій сумісності та мультипоточних алгоритмів за допомогою **NUnit** та **Moq**.
2.  **i18n (Internationalization)**: Повна підтримка української та англійської мов (ресурсні файли `.resx` на бекенді та JSON-словники на фронтенді).
3.  **Logging System**: Впроваджено **Serilog** для детального відстеження життєвого циклу програми, помилок та бізнес-подій у консоль та файли.
4.  **Microbenchmarking**: Реалізовано систему вимірювання продуктивності (`Stopwatch`) для аналізу ефективності паралельних алгоритмів.
5.  **External Libraries**: Інтеграція професійних інструментів: `CsvHelper` для парсингу даних, `FontAwesome` для іконографії інтерфейсу.
6.  **CI/CD & Automation**: Налаштовано **GitHub Actions** для автоматичного розгортання документації при кожному коміті.
7.  **Auto-Documentation**: Код повністю задокументований через XML-коментарі стандартного формату:
**[SkincareArchitect API Docs](https://dontwastemytimew.github.io/SkincareArchitect/)**

## Мультипоточне програмування (Lab 3a & 3b)

В рамках Лабораторної роботи №3 реалізовано паралельну систему обробки великих обсягів даних:
* **Lab 3a**: Порівняння послідовного та паралельного (`Parallel.ForEach`) алгоритмів парсингу 8494 косметичних засобів. Доведено значне прискорення обробки на багатоядерних системах.
* **Lab 3b**: Реалізація асинхронної клієнт-серверної архітектури. Обробка даних винесена в окремі потоки, що забезпечує стабільну роботу інтерфейсу (Non-blocking UI). Впроваджено Thread-safe механізми для запобігання Race Condition.
